# UX Notes - Camino de la Reconquista

## Objective
Build a scroll-driven storytelling experience around the historical route.

## Behavior
- The route progresses as the user scrolls.
- Each point becomes active when its text section enters view.
- The map camera pans and zooms to the active point.
- The route line is progressively revealed.
- The active point must be visually highlighted.
- Each point has an optional podcast/audio explanation.

## Desktop
- Split layout
- Left content panel
- Right sticky map
- Clean editorial UI

## Mobile
- Mobile-first
- Simple map view
- Point card / bottom sheet
- Clear CTA to play audio
- Easy to use during the physical tour

## Tone
- Historical
- Premium
- Editorial
- Cultural
- Minimal

## Avoid
- Generic tourism template
- Loud colors
- Overdecorated UI
- Clutter
